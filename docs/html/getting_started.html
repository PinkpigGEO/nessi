Getting started¶

NeSSI is currently developed under 64-bit GNU/Linux distributions only (Debian 9 Stretch and Ubuntu 16.10) using virtual environments created with Miniconda. NeSSI is designed to run with some of the major Python releases (2.7, 3.4, 3.5 and 3.6) since all users doesn’t have access to the latest version.

If you want to use a specific release of Python which is not supported by your OS, we strongly recommand to create a virtual environment using Miniconda instead of manually installing Python from sources on your system (Don’t suffer from Shiny New Stuff Syndrome - Don’t break Debian).

Dependencies¶

NeSSI requiered:

  • a C and a Fortran compiler such as GCC
  • the Numpy python package
  • the Scipy python package
  • the Matplotlib python package

All can be installed using your package manager and/or the Python pip package manager.

Installation¶

The installation of NeSSI is not automatized yet (with a setup.py for example) but it is quite simple.

In a terminal, go to the nessi/nessi folder and type: make .

Then, you have to add the path of the nessi package to PYTHONPATH. In your .bashrc file, add:

export PYTHONPATH=$PYTHONPATH:/path/to/the/nessi/package
